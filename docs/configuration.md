# Configuration reference

dne is configured via command-line flags. The Helm chart maps `values.yaml` settings onto the same flags; both surfaces are documented here.

## Flags

| Flag                            | Helm value               | Default              | Notes                                                                                              |
|---------------------------------|--------------------------|----------------------|----------------------------------------------------------------------------------------------------|
| `--namespaces`                  | `namespaces`             | (empty)              | Comma-separated namespace list. Empty = cluster-wide. Applied at the watch layer.                  |
| `--label-selector`              | `labelSelector`          | (empty)              | Standard k8s label selector. Empty = no filter.                                                    |
| `--metrics-bind-address`        | `metricsPort` (port)     | `:8080`              | Address for `/metrics`. Set to `:0` to disable.                                                    |
| `--health-probe-bind-address`   | `healthPort` (port)      | `:8081`              | Address for `/healthz` and `/readyz`.                                                              |
| `--leader-elect`                | `leaderElection.enabled` | `false`              | Enable leader election. Required if you run more than one replica.                                  |
| `--leader-election-id`          | `leaderElection.id`      | `dne.dgamo.github.io`| Lease name in the release namespace.                                                                |
| `--log-level`                   | `logLevel`               | `info`               | `debug`, `info`, `warn`, `error`.                                                                  |
| `--version`                     | n/a                      | n/a                  | Print the binary's version and exit.                                                                |

## Scope filters

`namespaces` and `labelSelector` are independent and combine with AND: when both are set, only Secrets in the listed namespaces that also match the selector are watched.

Both filters run at the watch layer (`cache.Options.DefaultNamespaces` and `cache.Options.ByObject.Label`), so out-of-scope Secrets never enter the controller's cache. This is bandwidth-efficient on large clusters and ensures that removing a label triggers a synthetic delete event — which causes dne to clean up the corresponding metric series automatically.

Recommended label-selector convention: tag the Secrets you want dne to track with `dne.k8s.io/watch=true`, then run:

```bash
helm upgrade dne dne/dne --reuse-values --set labelSelector="dne.k8s.io/watch=true"
```

Or reuse a label you already maintain, e.g. `app.kubernetes.io/component=tls`.

## RBAC

dne requires `secrets: [get, list, watch]` cluster-wide, even when `--namespaces` is set. Per-namespace `Role` objects are not generated by the chart — it would multiply the number of chart objects without improving the security posture, because dne still authenticates as a single ServiceAccount.

If you need strict namespace isolation, deploy dne separately in each namespace with a `Role` instead of a `ClusterRole`. That use case is intentionally out of scope for the default chart.

When `leaderElection.enabled=true`, the chart additionally grants `coordination.k8s.io/leases: [get, create, update, patch]` and `events: [create, patch]`.

## Replicas, leader election, and HA

The default is `replicaCount: 1` with leader election off, because:

- dne is stateless. A restart picks up where it left off within seconds (cache relist).
- Two replicas without leader election would both publish identical series under different `instance` labels — not a Prometheus error, but doubles the scrape cost.
- A brief outage during a restart means slightly stale `_not_after_seconds` values, which are still correct enough to alert on.

If you really want HA:

```bash
helm upgrade dne dne/dne --reuse-values \
  --set replicaCount=2 \
  --set leaderElection.enabled=true
```

## Extra args / environment

Use `extraArgs` and `env` (both arrays in `values.yaml`) for anything not covered by an explicit value. Example:

```yaml
extraArgs:
  - "--health-probe-bind-address=:9000"
env:
  - name: GOMAXPROCS
    value: "1"
```

## Endpoints

| Path        | Default port | Purpose                              |
|-------------|--------------|--------------------------------------|
| `/metrics`  | `8080`       | Prometheus exposition.               |
| `/healthz`  | `8081`       | Liveness probe.                      |
| `/readyz`   | `8081`       | Readiness probe.                     |

Probes are wired up in the Deployment template; the `/healthz` and `/readyz` handlers are controller-runtime's `healthz.Ping` (returns 200 once the manager is started).
